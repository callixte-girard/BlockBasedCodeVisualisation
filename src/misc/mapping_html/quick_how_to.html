• one-level (small) blockly tags
    ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
    quick helps to what you want to get in your assembling
    ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
    • make blocks
      • block with a value in it
        • <block type={blockly_type}>
          • {block content = attributes and children}
        • </block>
      • floating block with a value in it
        • <shadow type={blockly_type}>
          • {block content = attributes and children}
        • </shadow>
    • statements
      • set a value in a variable @VariableDeclaration @AssignmentExpression
        • <value name="VALUE">
          • {block that contains the value to assign}
        • </value>
      • if / else if / else... @IfStatement
        • <value name="
          • IF0 (if)
          • IF1/2/3... (else if)
          • ELSE (else)
        • ">
          • {condition block #boolean }
        • </value>
      • set the values for a for loop (from , to , by) @ForStatement
        • <value name="FROM">
          • {block of the init expression}
        • </value>
        • <value name="TO">
          • {block of the limit expression}
        • </value>
        • <value name="BY">
          • {block of the update expression}
        • </value>
      • set while | until condition @WhileStatement
        • <value name="BOOL">
          • {condition}
        • </value>
      • set while | until mode @WhileStatement
        • <field name="MODE">
          • WHILE
          • UNTIL
        • </field>
      • (if condition) ...do these things @Statement
        statement['body'] or sthng like this
        • <statement name="
          • DOn
            accordingly to corresponding IFn. if for | while, just put "DO".
          • ELSE
            has no IFn associated
        • ">
          • {instructions}
        • </statement>
    • expressions
      • set the left member of an expression evaluation @BinaryExpression
        • <value name="A">
          • {block of the left expression}
        • </value>
      • set the right member of an expression evaluation @BinaryExpression
        • <value name="B">
          • {block of the right expression}
        • </value>
      • set the operation performed to an expression (logical or arithmetic) @LogicalExpression @BinaryExpression
        • <field name="OP">
          • {initials of the operation}
            see specific section for more info
        • </field>
    • end values
      • name of a variable @Identifier
        • <field name="VAR">
          • { @varName }
        • </field>
      • end slave value @Literal
        • <field name="
          • NUM #number
          • TEXT #string
          • BOOL #boolean
        • ">
          • { @varValue }
        • </field>
